动态规划? 错 是动态递推

动态规划

动态规划 跟 "递归 分治 回溯" 没有太大的区别 我们来看看动态规划的具体定义

1. simplifying a complicated problem by breaking it down into simpler sub-problems
	这个是 wiki百科 对动态规划的解释
	动态规划 英文我个人觉得翻译最好的为 "动态推导/动态递推"(也是之前 双指针详解 的思想, "咬文嚼字" 总归是好的)
	上述英文的意思为 "把一个复杂的问题分解成更简单的子问题"
	其实就是分治的思想 但是 分治 跟 动态规划 只有小小的不一样 那就是 第二点

2. 动态规划 = 分治 + 最优子结构
	一般来说 动态规划 会让你求 最优解 最大值 最小值
	正因为 他有所谓的最优子结构 所以 你就不需要 将中间所有的状态 都保存下来 只需要存最优的状态

***重点总结:
	动态规划 和 递归 分治 没有根本上的区别 (关键看有无最优子结构)
  共性: 找到重复子问题
  差异性: 最优子结构、中途可以淘汰最优解

例题1 - 爬楼梯 - 斐波那契数列

别急 我们先来看看 斐波那契数列

斐波那契数列 我们都知道是可以用递归去解决
	但是面试的时候 千万不要说用递归 啥递归的效率是极慢的 时间复杂度 为 指数级
	如果是 上述的图的斐波那契数列 则是O(2^n)

所以我们应该怎么优化呢
	我们可以加一个缓存 可以用数组 这种叫做 "记忆化搜索"

// golang 代码示范
func fib(n int, memo []int) {
  // resursion terminator(递归终止条件)
  if (n <= 1) {
    return 0
  } 
  // 
  if (memo[n] == 0) {
    memo[n] = fib(n - 1) + fib(n - 2)
  }
  return memo[n]
}

/*
	我们通过 增加 数组实现的	缓存
	使得只要在数组出现的值就直接return 不参与遍历  把重复的节点直接砍掉
	所以 O(2^n) 变成了 O(n)
	其实这个也是分治的思想 只不过 加了个记忆化搜索
*/ 
// golang
a[0] = 0
a[1] = 1
for (i := 2; i <= n; i++) {
  a[i] = a[i-1] + a[i-2]
}

/*
	我们可以利用这个循环 先把头两个元素定义好
	随后直接for循环 最后a[n] 就是结果
	
	这种的思想 也就是 自底向上 的思想
*/

自顶向下

我们当然也可以用 分治+记忆化搜索 来实现 这个思想是 "自顶向下" 的思想
什么叫自顶向下呢 就是看上面图 我们一直从上面找到最下面的叶子结点
比如:
	我要算fib[6] 
		就得算fib[5] 和 fib[4]
	想算fib[5]
		就得算fib[4] 和 fib[3]
	一直下去 中间算过的结果 我们可以用 记忆化搜索
	符合人脑的习惯 也是分治的思想

自底向上

但是 后来当我们面对更难得dp 包括那些高手 遇到dp题目 直接会用 "自底向上" 的思想
什么是自底向上呢
比如:
	我们想算 fib[6]
		从fib[0] fib[1] 开始
	然后一直用循环来递推

/*
***所以 我们只需要了解 自顶向下 即可 我们遇到dp题目 直接用 "自底向上" 当然其实还是无所谓 最好是 自底向上
因为只要是专业的 绝对是自底向上
所以动态递归 最好翻译成 动态递推
*/

我们来看看 真正的例题 帮我们去了解 自底向上 的思想

https://leetcode.cn/problems/climbing-stairs/

既然分析完 我们来试试例题
这是一个 很经典的爬楼梯的问题
大家先去看完题目
如果有两层楼梯 我们只能这么走
1. 一层 -> 一层
2. 二层

如果有三层楼梯 我们只能这么走
1. 一层 -> 一层 -> 一层
2. 一层 -> 两层
3. 两层 -> 一层

我们其实发现 这个其实就是斐波那契数列

class Solution {
public:
    int climbStairs(int n) {
        /* 定义dp数组 */
        vector<int> dp(n+1);
        /* 初始化dp数组 */
        dp[0] = 1;
        dp[1] = 1;
        /* dp方程 自底向上 */
        for (int i = 2; i < dp.size(); i++) {
            dp[i] = dp[i-1] + dp[i-2];
        }
        return dp[n];
    }
};

例题2 - 路径计数 - 二维数组

https://leetcode.cn/problems/unique-paths/

像 例题1 是一维的 那么 dp题目 更难得也有 二维的 中间也有取舍最优子结构了 就比如这个题目

1. 我们 直接 采用 "自底向上" 的编程思想	
	所以我们 得用 一个 储存中间状态的数组 这里是二维数组 opt[i][j]

2. 随后我们来观察题目 
	由于我们每一步只能从向下或者向右移动一步
	因此要想走到 (i,j)(i,j)
	如果向下走一步，那么会从 (i−1,j)(i−1,j) 走过来
	如果向右走一步，那么会从 (i, j-1)(i,j−1) 走过来
	因此我们可以写出动态规划转移方程：dp[i,j] = dp[i−1,j] + dp[i,j−1](目前不知道啥叫转移方程 不要紧 先别管)

3. 如果 i = 0 相当于只有一行的路径所以 dp[0][j] = 1;
	如果 j = 0 相当于只有一列的路径所以 dp[i][0] = 1;

例题2 - 题解

执行用时：0 ms, 在所有 C++ 提交中击败了100.00%的用户
内存消耗：6.2 MB, 在所有 C++ 提交中击败了61.26%的用户
  
class Solution {
public:
    int uniquePaths(int m, int n) {
        /* 定义dp数组 */
        vector<vector<int>> dp(m, vector<int>(n));
        /* 初始化dp数组 */
        for (int i = 0; i < m; i++) dp[i][0] = 1;
        for (int i = 0; i < n; i++) dp[0][i] = 1;
        /* dp方程 */
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) dp[i][j] = dp[i-1][j] + dp[i][j-1];
        }
        return dp[m-1][n-1];
    }
};

/*
  时间复杂度 O(m*n)
  空间复杂度 O(m*n)
  但是，您可能已经注意到
  每次我们更新时dp[i][j]，我们只需要dp[i - 1][j](在上一行)和dp[i][j - 1](在当前行)
  所以我们可以将内存使用量减少到只有两行(O(n))
*/ 
执行用时：0 ms, 在所有 C++ 提交中击败了100.00%的用户
内存消耗：6.3 MB, 在所有 C++ 提交中击败了56.35%的用户

class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<int> pre(n, 1), cur(n, 1);
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                cur[j] = pre[j] + cur[j - 1];
            }
            swap(pre, cur);
        }
        return pre[n - 1];
    }
};

/*
  这个是二维数组 我们其实可以把这个降维 因为可以省内存空间
  pre[j] 和 cur[j]在更新之前
*/ 
执行用时：0 ms, 在所有 C++ 提交中击败了100.00%的用户
内存消耗：6 MB, 在所有 C++ 提交中击败了69.52%的用户

class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<int> cur(n, 1);
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                cur[j] += cur[j - 1];
            }
        }
        return cur[n - 1];
    }
};

状态转移方程(DP方程)

讲了 两道例题 我们在分析一下

两道例题 其实都有一个核心的关键方程 
例题1 是 v[i] = v[i-1] + v[i-2]
例题2 是 v[i][j] = v[i-1][j] + v[i][j-1]

这类方程 我们叫状态转移方程 也叫dp方程 但是又回到 咬文嚼字上 这其实就是一个递推方程

动态规划关键点&思考步骤

关键点:
  1. 最优子结构 opt[n] = best_of(opt[n-1], opt[n-2])
  2. 储存中间状态 opt[i]
  3. 递推公式(状态转移方程(DP方程))
  Fib: opt[i] = opt[n-1] + opt[n-2]
  二维路径: opt[i, j] = opt[i-1][j] + opt[i][j-1]

	// 对于初学者 面试 来说 最重要其实第二步 但是 对于比较难的dp 来说 最重要是第三步 应该很难找到合适的公式

思考步骤:
	1. 化繁为简 将 复杂的问题 变成子问题 其实就是 找重复性(分治思想)
	2. 定义好状态空间
		2.1 定义 dp 数组 && 初始化 dp 数组
		2.2 注意这里 定义 和 初始化 长度容积 一定要注意 有时候会是 ">=" 或者其他的 如果定义错误 则会数组越界 或者 答案错误
	3. 写好 dp方程

例题3 - 不同路径II - 二维数组

我们看了两道例题后 我们再来 锻炼锻炼

这道题其实 跟例二思路 是一样的 就是多了一个障碍物
其实 这个障碍物 很好去解决 如果 obstacleGrid[i][j] == 1 也就是碰到障碍区 直接跳过就是了

例题3 - 题解

执行用时：0 ms, 在所有 C++ 提交中击败了100.00%的用户
内存消耗：7.6 MB, 在所有 C++ 提交中击败了31.87%的用户

class Solution {
public:
    int uniquePathsWithObstacles(vector<vector<int>>& obstacleGrid) {
        /* 计算数组长度 */
        int m = obstacleGrid.size();
        int n = obstacleGrid[0].size();
        /* 定义数组 */
        vector<vector<int>> dp(m, vector<int>(n));
        /* 初始化数组 */
        for (int i = 0; i < m && obstacleGrid[i][0] == 0; i++) dp[i][0] = 1;
        for (int i = 0; i < n && obstacleGrid[0][i] == 0; i++) dp[0][i] = 1;
        /* dp方程 注意有障碍的情况 */
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (obstacleGrid[i][j] == 1) continue;
                dp[i][j] = dp[i-1][j] + dp[i][j-1];
            }
        }
        return dp[m-1][n-1];
    }
};

例题4 - 题解

执行用时：20 ms, 在所有 C++ 提交中击败了75.36%的用户

class Solution {
public:
    int longestCommonSubsequence(string text1, string text2) {
        /* 定义数组 */
        vector<vector<int>> dp(text1.size()+1, vector<int>(text2.size()+1));
        
        /* dp方程 */ 
        for(int i = 1; i <= text1.size(); i++){
            for(int j = 1; j <= text2.size(); j++){
                if(text1[i - 1] == text2[j - 1]){
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                }
                else{
                    dp[i][j] = max(dp[i - 1][j],dp[i][j - 1]);
                }
            }
        }
        return dp[text1.size()][text2.size()];
    }
};
