双指针

在我们刚开始刷算法 不可能直接干树 递归 甚至动态规划这些 我们肯定是先从数组入手

对于数组来说 双指针 其实是一个常用的解法 对于链表来说也是 
双指针也是分很多种类的 在接下来会讲解

数组-左右指针

其实快慢指针 也有被说左右指针 但是本人认为这么说 其实是相当不合理 在此之上 我便有了一丝 "咬文嚼字" 的感觉
但是我始终认为 你给一个方法取名 其实就是代表了 你对这个方法的理解

所以我也很推荐你们能拥有自己的想法 不要在意别人的命名 就好比 leetcode 题解 就有把快慢指针说为左右指针
你能说他错吗 其实快慢指针 也确实就是一个指针左 一个指针右 哈哈哈 但是这么说快慢指针为左右指针 非常肤浅

从代码和图里 我们也能看到 左右指针 本人对他的理解 则是 左指针为数组或者链表的开头 右指针则为数组的最后一个索引的元素或链表的tail

所以左指针满足一定的条件 会往右移动 右指针满足一定的条件 会往左移动 直到 right < left 停止

双指针的解题思路

双指针的应用场景

双指针的应用场景也是在数组或者链表 中 找到一个元素的子集 该子集满足某种限制 甚至这个子集也是一个数组

***首先要说明 子集 的定义 数组的子集 就是 这个数组里面所有的元素任意去组合 甚至 这个数组的本身也是这个数组的子集

例题一

https://leetcode.cn/problems/move-zeroes 移动0

我们在看完这题后 先尝试用暴力求解 就是不要去考虑什么谈情说爱 啊呸 考虑什么时间复杂度 空间复杂度 做出来就行

然后我们来看双指针 这个解法 先把答案贴在下面

// c++
class Solution {
  public:
  void moveZeroes(vector<int>& nums) {
    int slow = 0;
    for (int fast = 0; fast < nums.size(); ++fast) {
      if (nums[fast] != 0) {
        swap(nums[slow], nums[fast]);
        slow++;
      };
    }
  }
};
// golang
func moveZeroes(nums []int)  {
    // 双指针 --- 快慢指针
    slow := 0
    for fast := 0; fast < len(nums); fast++ {
        // 根据具体内容写相应代码 
        if nums[fast] != 0 {
            nums[slow], nums[fast] = nums[fast], nums[slow]
            slow++
        }
    }
}
思路讲解

我们看到这道数组 他需要将一个数组 nums 里面元素为0的全部移到后面
一个数组的子集 在这道题 子集其实也就是数组的本身 然后他需要满足0都要在最后的一个限制 所以这道题完美应用 双指针的应用场景
