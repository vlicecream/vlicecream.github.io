UE4 入门概念知识

UE4 v4.27.2 目录结构说明

- config
  里面都是 ini 配置文件夹 记录着我们的配置  !不要动
  
- content
  和项目中是一一对应的
  - Collections 是存放临时文件 
  - Developers 是存放临时文件 
  	也是不要去改动
  - StarterContent
  	- ... 
  		里面存放的都是资源 高版本兼容低版本资源 反之则不行
  
- Intermediate
  这个如果项目完成了 我们是可以删除他的 因为是引擎使用中的中间件
  
- Saved
  这个不建议 删除 虽然存放的也是临时文件 但是不好恢复
  
---------------------------------- 以上是纯蓝图 会有的文件 c++ 会多出来几个文件夹----------------------------------

- .vs c++ 解决方案的文件 智能提示
- xxx.sln
- Binaries 二进制文件 编译器热加载c++类的文件
- Source 放着c++ 的一个源文件

Unreal缓存数据解读

1. 保管库
  我们 保管库会越来越大 只需要在设置里面的设置路径即可

  我们升级版本的时候 保管库也会创建一个文件夹 如果我们用的版本高 低版本甚至引擎都没了 建议删除引擎的时候 把对应保管库也删了

  确定保管库用不着的时候 也可以删除的 反正在下载下来即可
  
2. 安装目录 windows 为例
  c盘 -> 用户 -> Bathur
  随后在路径后面直接写 AppData
  -> local -> UnrealEngine
  这里是一个很隐秘的存放材质包的目录

虚幻引擎的三层设置简略介绍

偏好设置
  偏好设置一般就是设置一些编辑器的外观 字体 跟实际项目其实没太大的关系
项目设置
  可以设置 项目的启动目录 还有游戏模式 等等
世界设置
  可以设置 灯光 游戏规则 等等

编辑器窗口布局

1. 可以灵活拖拽
2. 如果窗口是单标签 可以邮件隐藏 点击黄色区域后就可以显示出来
3. 如果不小心删掉 可以从window可以再次显示出来
4. 可以从window在拖拽一个重复的窗口 方便查看
5. 只要是斜边就可以放到最顶层使用 对于某一类游戏资源 都是斜边编辑器
6. 默认窗口 可以直接window里找到重启 即可恢复

c++与蓝图的关系

- 两者关系
  - 蓝图建立在c++代码之上
  - 蓝图与c++可以很好的协作
  
- 理论平衡
  - c++ 由游戏逻辑程序员使用
  	· 完成虚幻尚未封装的功能
  	· 完成根据项目需求需要自定义的功能
  - 蓝图由游戏设计人员使用
  	· 设计其他游戏资源
    · 功能测试时使用
  	· 项目快速迭代时使用
  	· 调用编写好的c++代码

什么是蓝图可视化脚本

蓝图可视化脚本 简称 "蓝图"、 "蓝图脚本"
  
蓝图是一种可视化编程
  实际上 你使用蓝图的时候就是在编程
  蓝图是一种需要编译的面向对象的可视化编程语言
  
蓝图完全集成在虚幻4中
  
通过节点与连线工作

什么是蓝图系统

蓝图系统俗称 "蓝图类"
  
将蓝图类想象成游戏内容的容器
  可以包含脚本
  可以包含组件
  可以仅仅包含数据
