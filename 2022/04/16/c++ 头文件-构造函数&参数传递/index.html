c++ 头文件 构造函数&参数传递

开文说明

本文章大多数都是根据以下代码为模范讲解

class complex {
public:
  complex(double r = 0, double i = 0) : re(r), im(i) {}
  complex& operator+=(const complex&);
  double real() const { return re; }
  double imag() const { return im; }

private:
  double re, im;

  friend complex& __doapl(complex*, const complex&);
};

构造函数

What?

​ 如果你想要创建一个对象，有一个函数会自动调用起来，这个就是构造函数

{
  complex c1(2, 1);  // 有参数的创建对象
  complex c2;  // 无参数的创建对象
  complex*p = new comolex(4);  // 动态的创建对象 是一个指针
}

Tips:

​ 构造函数的名称一定要跟类的名称一致

​ 构造函数是有参数的

​ 参数可以为默认的

​ 返回值是没有指定类型的 也不需要写

​ re(r) im(i) 是初始列 因为我们在这里面将r i初始化 等同于 在接下来的大括号写{ re = r; im = i;}

complex(double r = 0, double i = 0) : re(r), im(i) {}
// r = 0 i = 0 都是设置了默认参数
// re(r) im(i) 是初始列 不需要指定类型

// ps:上述的一行可以写成 但是推荐上述写法 在初始列 里面将re im 初始化
complex(double r = 0, double i = 0) { re = r; im = i; }

Overloading(重载)

构造函数可以有很多个

如果你在class body里定义了

double real () const { return re; }

又在外部定义了

void real(double r) { re = r; }

这两个是不冲突的 因为编译器会把他们换成不一样的名字

那么我们来看看冲突的

complex (double r = 0, double i = 0) : re (r), im (i) { }
complex (): re(0), im(0) { }

上述两个不是不行的 因为 如果这么调用

{
  complex c1;
  complex c2();
}

这个时候他就会分不清到底要调用哪一个 complex 所以就会报错

当构造函数在private区域里

构造函数在private里面我们无法通过正常的模式构造对象

所以一般我们是不会把构造函数放在私有区域

有一个设计模式 Singleton 会把构造函数放在private里

Const(常量成员函数)

double real () const { return re; }
作用：代表 数据内容 不可改变    

                                                                                                                                                                                                                                                                                                          

参数传递

By value: 值传递 把这个值都传过去 最好别用

By reference: 指针传递 传指针 一直都会是8 btye 最好传指针

Complex& operator += { const complex& }

像这个其实就是 by reference

complex (double r = 0, double i = 0): re (r), im (r) { }

这个就会是 by value

返回值传递

跟参数传递一致 最好使用指针传递

friend(友元)

private:
	double re, im;
	friend complex& __dopal (conplex*, const complex&)
inline complex& __doapl (complex* ths, const complex& r) {
  ths->re += r.re;
  ths->im += r.im;
  return *ths
}

Friend的意义在于打开封装的大门 可以直接从私有区域拿数据

就好比 借钱 陌生人不可能借钱给他 但是朋友可以考虑 但是朋友太多借太多钱也不好 所以凭你自己选择 如果不设置friend也可以 绕个弯从函数拿 只是慢点

相同class的各个obj互为friend

class complex {
public:
  complex(double r = 0, double i = 0) : re(r), im(i) {}
  
  int func(const complex& param) { return param.re + param.im; }

private:
  double re, im;
};

{
  complex c1(2, 1);
  complex c2;
  
  c2.func(c1);
}

在上述代码中 我们竟然可以通过func直接获取到private中的值 可是我们可是也没有设置friend

因为 (看小标题)

什么情况下不需要用到引用传递

总结代码规范

1.数据一定放在private

2.数据&返回值尽量以reference传递

3.const应该加就要加

4.构造函数的初始列
