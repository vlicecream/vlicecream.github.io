一、模板语法传值

1. 什么是模板语法传值
	我们在后端写的python数据类型可以利用模板语法传到前端
    """
    ps: 经验之谈  {{}}:变量相关    {%%}:逻辑相关
    """
 
2. 代码演示
	# 后端代码
    	from django.shortcuts import render
        # Create your views here.
        def index(request):
            s = 'yxx想奔现'
            i = 123456
            f = 88.88
            l = ['叶叶', '晨晨', '诚诚']
            b = True
            t = (111, 222, 333, 444)
            d = {'username': 'jason', 'age': 18, 'info': '这个人有点意思'}
            se = {'晶晶', '瑶瑶', '嘤嘤'}

            def func():
                print('你好')
                return 'func'

            class Myclass():
                def get_self(self):
                    return 'self'

                @staticmethod
                def get_func():
                    return 'get_func'

                @classmethod
                def get_class(cls):
                    return 'get_class'
                
			   # 对象被展示到html页面上 就类似于执行了打印操作也会触发__str__方法
                def __str__(self):
                    return '奔现奔现'

            obj = Myclass()

            return render(request, 'index.html', locals())
        
        # 前端语法
            <p>{{ n }}</p>
            <p>{{ f }}</p>
            <p>{{ s }}</p>
            <p>{{ b }}</p>
            <p>{{ l }}</p>
            <p>{{ d }}</p>
            <p>{{ t }}</p>
            <p>{{ se }}</p>
            <p>传递函数名会自动加括号调用 但是模版语法不支持给函数传额外的参数:{{ func }}</p>
            <p>传类名的时候也会自动加括号调用(实例化){{ MyClass }}</p>
            <p>内部能够自动判断出当前的变量名是否可以加括号调用 如果可以就会自动执行  针对的是函数名和类名</p>
            <p>{{ obj }}</p>
            <p>{{ obj.get_self }}</p>
            <p>{{ obj.get_func }}</p>
            <p>{{ obj.get_class }}</p>

二、模板语法之过滤器

1. 什么是模板语法过滤器
	过滤器就是类似于模板语法内置的内置方法
    django内置有60多个过滤器 我们不需要学这么多 了解10个左右就差不多了 后面碰到了再去记忆
    ps: 过滤器最多只有两个参数
    
2. 基本语法
	{{数据|过滤器:参数}}
    
3. 转义
	# 前端
		|safe
	# 后端
		from django.utils.safestring import mark_safe
  			res = mark_safe('<h1>新新</h1>')
    """
    以后你在全栈项目的时候 前端代码不一定非要在前端页面书写
    也可以现在先在后端写好 然后传递给前端页面
    """
    
4. 过滤器介绍
    <h1>过滤器</h1>
    <p>统计长度:{{ s|length }}</p>
    <p>默认值(第一个参数布尔值是True就展示第一个参数的值否在展示冒号后面的值):{{ b|default:'啥也不是' }}</p>
    <p>文件大小:{{ file_size|filesizeformat }}</p>
    <p>日期格式化:{{ current_time|date:'Y-m-d H:i:s' }}</p>
    <p>切片操作(支持步长):{{ l|slice:'0:4:2' }}</p>
    <p>切取字符(包含三个点):{{ info|truncatechars:9 }}</p>
    <p>切取单词(不包含三个点 按照空格切):{{ egl|truncatewords:9 }}</p>
    <p>切取单词(不包含三个点 按照空格切):{{ info|truncatewords:9 }}</p>
    <p>移除特定的字符:{{ msg|cut:' ' }}</p>
    <p>拼接操作:{{ l|join:'$' }}</p>
    <p>拼接操作(加法):{{ n|add:10 }}</p>
    <p>拼接操作(加法):{{ s|add:msg }}</p>

四、自定义过滤器 标签 INCLUSION_TAG

1. 必须记住的三步曲
	1.1 在应用下面创建'templatetags' 文件夹
    1.2 然后在这个文件夹内创建任意名字.py文件
    1.3 最后在文件里面输入 
    	from django import templa  
        register = templa.Library()
       
    
2. 自定义过滤器
	# 代码演示
        @register.filter(name='baby')
        def my_sum(v1, v2):
            return v1 + v2
        # 前端
        {% load mytag %}
        <p>{{ n|baby:666 }}</p>
        
3. 自定义标签 (参数可以有多个) 类似于自定义函数
	# 代码演示
        @register.simple_tag(name='plus')
        def index(a,b,c,d):
            return '%s-%s-%s-%s'%(a,b,c,d)
        # 使用
        标签多个参数彼此之间空格隔开
        <p>{% plus 'jason' 123 123 123 %}</p>

4. 自定义inclusion_tag
    """
    内部原理
        先定义一个方法 
        在页面上调用该方法 并且可以传值
        该方法会生成一些数据然后传递给一个html页面
        之后将渲染好的结果放到调用的位置
    """
    @register.inclusion_tag('left_menu.html')
    def left(n):
        data = ['第{}项'.format(i) for i in range(n)]
        # 第一种
        # return {'data':data}  # 将data传递给left_menu.html
        # 第二种
        return locals()  # 将data传递给left_menu.html
