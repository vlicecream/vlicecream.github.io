2.异步  在提交了任务请求后，不在原地等着拿任务执行结果，直接执行下一个任务请求

'''阻塞与非阻塞'''
1.阻塞     阻塞状态
2.非阻塞   运行状态

理想状态：时时刻刻让程序保持运行状态

八、僵尸进程与孤儿进程(了解)

"""僵尸进程"""
	僵尸进程就是死了没死透，依然还存在等着子进程去执行完，所有的进程都会变成僵尸进程，父进程运行完后还得观察子进程的各种信息，比如PID，子进程的运行时间，而且你运行了子程序后，子程序死后不会回收自己的PID，所以是父进程去帮忙回收
    子进程结束后 父进程回去回收子进程的PID
    或者就用join方法来回收子进程的PID
"""孤儿进程"""
    孤儿进程就是子进程存活，父进程意外死亡
    这个时候操作系统会生成一个“儿童福利院”来回收这些子进程的PID

# 大家有没有碰到这样的情况，就是在买票的时候明明还有票，可能过一会来看就没有票了，或者就是还剩下一张票的时候抢，发现没有票了，我们来模仿这个案例，来引出互斥锁
    import time
    import json
    import random
    from multiprocessing import Lock
    from multiprocessing import Process


    # 查看票数
    def check_ticker(i):
        with open('ticket_data.json', mode='rt', encoding='utf8') as file:
            file = json.load(file)
            print(f"用户{i}查询票还剩下{file['ticket']}")


    # 购买票
    def buy_ticket(i):
        # 查票
        with open('ticket_data.json', mode='rt', encoding='utf8') as file:
            file = json.load(file)
            # 模仿网络延迟
        time.sleep(random.randint(1, 4))
        if file.get('ticket') > 0:
            file['ticket'] -= 1
            with open('ticket_data.json', mode='wt', encoding='utf8') as f:
                json.dump(file, f)
            print(f"用户{i}购买成功")
        else:
            print(f'用户{i}购买失败')


    def run(i, mutex):
        check_ticker(i)
        # 生锁
        mutex.acquire()
        buy_ticket(i)
        # 解锁
        mutex.release()


    if __name__ == '__main__':
        # 在主进程中生成一把锁 让所有的子进程抢 谁先抢到谁先买票
        mutex = Lock()
        for i in range(1,11):
            p = Process(target=run, args=(i, mutex))
            p.start()

    import time
    import random
    from multiprocessing import Process
    from multiprocessing import Queue


    def producer(name, food, q):
        for i in range(10):  # 生产9次
            data = f'{name}生产了{food}{i}'

            print(f'{name}把{food}都吃了')


    if __name__ == '__main__':
        q = Queue()
        bot1 = Process(target=producer, args=('1', '泔水', q))

        eater = Process(target=consumer, args=('3', q))
        bot1.start()
        bot2.start()
        eater.start()
        bot1.join()
        bot2.join()
        q.put(None)
        q.put(None)

"""线程互斥锁"""
import time
from threading import Thread
from threading import Lock

money = 100
mutex = Lock()


def func():
    global money
    mutex.acquire()
    tmp = money
    time.sleep(0.1)
    money = tmp - 1
    mutex.release()


if __name__ == '__main__':
    q_list = []
    for i in range(100):
        q = Thread(target=func)
        q.start()
        q_list.append(q)
    for q in q_list:
        q.join()
    print(money)

二十三、GIL与普通互斥锁的区别

from threading import Thread,Lock
import time


mutex = Lock()
money = 100


def task():
    global money
    # with mutex:
    #     tmp = money
    #     time.sleep(0.1)
    #     money = tmp -1
    mutex.acquire()
    tmp = money
    time.sleep(0.1)  # 只要你进入IO了 GIL会自动释放
    money = tmp - 1
    mutex.release()


if __name__ == '__main__':
    t_list = []
    for i in range(100):
        t = Thread(target=task)
        t.start()
        t_list.append(t)
    for t in t_list:
        t.join()
    print(money)



"""
100个线程起起来之后  要先去抢GIL
我进入io GIL自动释放 但是我手上还有一个自己的互斥锁
其他线程虽然抢到了GIL但是抢不到互斥锁

二十五、死锁现象

from threading import Thread
from threading import Lock
import time

musex1 = Lock()
musex2 = Lock()


class MyThread(Thread):
    def run(self):
        self.run1()
        self.run2()

    def run1(self):
        musex1.acquire()
        print(f'{self.name}抢到了1锁')
        musex2.acquire()
        print(f'{self.name}抢到了2锁')
        musex2.release()
        musex1.release()

    def run2(self):
        musex2.acquire()
        print(f'{self.name}抢到了2锁')
        time.sleep(3)
        musex1.acquire()
        print(f'{self.name}抢到了1锁')
        musex1.release()
        musex2.release()


if __name__ == '__main__':
    for i in range(10):
        q = MyThread()
        q.start()
"""
	这就是死锁，你运行了就会发现程序一直在卡死(阻塞)，我们来看原因，一开始第一个线程去抢1锁，抢到以后其他线程就只能看着，然后一线程抢2锁的时候肯定无忧无虑，因为其他线程还在外面待着，然后1线程解了2锁和1锁，这个时候有一个线程一进抢了1锁，1线程又去run2抢了2锁，这时候就尴尬了，2线程要2锁，1线程要1锁，就这成了死锁
"""

三十一、线程池

from concurrent.futures import ThreadPoolExecutor
import time

pool = ThreadPoolExecutor(5)  # 括号内的参数可以不传 默认是开启计算机cpu个数的5倍的线程 写了5池子里就是开启5个线程
"""
池子造出来后，里面会固定存在5个线程，这个五个线程不会重复创建和销毁
"""


def task(n):
    print(n)
    time.sleep(2)
    return n ** n


l_list = []
for i in range(20):
    """
    讲到提交任务就会想到同步异步两种方法
    同步：提交完任务在原地等数据结果的到来，不会离开
    异步：提交完任务便执行下一步代码，不会原地等待数据结果的到来
    """
    res = pool.submit(task, i)  # 向池子提交任务即可
    l_list.append(res)

pool.shutdown()  # 关闭线程池 等待线程池里面的线程所有的任务运行完毕
for t in l_list:
    print('---->', t.result())
    """
    result 会让程序并发变成串行，并且返回异步提交任务的返回结果
    """

    print(n)

    print('----->', n.result())

    for i in range(20):

        """
        res = pool.submit(task, i).add_done_callback(callback)
