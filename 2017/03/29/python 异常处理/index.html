python 异常处理

异常处理

# 什么是异常
    异常就是你程序报错，导致了程序中途终止
   
# 为什么要处理异常
    为了程序的健壮性，不会轻易报错停止
    将发生的异常写入日志
   
# 怎么处理异常
    1.语法上的异常
        a = 1
        b = 0
        if a>b
            print('123')
        '''
            if a>b
                 ^
        SyntaxError: invalid syntax
        '''
    # 这种就是语法上的问题，那我们应该怎么处理呢，是直接修改正确，这种愚蠢的错误，不允许发生

    2.逻辑上的异常
    2.1 可以预测到的逻辑上的异常
        age = input('----->')
        age = int(age)
        if age == 18:
            print('恭喜你猜对了')	
   '''
   这种的就是能预测到的异常，万一用户zz偏要输入字符串怎么办呢，那我们应该怎么解决这个，既然能预测到了，就直接增加判断去解决其问题，不要用异常处理，为什么呢，稍后为你揭秘
   '''
    age = input('----->')
    if age.isdigit():
        age = int(age)
        if age == 18:
            print('恭喜你猜对了')
    else:
        print('必须输入数字')
        
   2.2 不可以预测到的逻辑上的异常
    # 在后面的套接字中我们会遇到一个不可以预测到的逻辑上的异常，在软件与服务器之间的传输信息中，我们是预测不到发生什么错误的，所以就得去异常处理
    # 异常处理模板
    try:
        子代码块  # 有可能抛出异常的代码块
    except 异常类型 as 变量名:
        pass
    except 异常类型1 as 变量名1:
        pass
    except 异常类型2 as 变量名2:
        pass
    else:
        pass  # 这个是如果这个子代码块没有抛出异常，就会执行的代码
    finally:
        pass  # 这个是无论这个子代码块有没有抛出异常，都会执行的代码
    '''
    那么这个异常处理是降低了代码可读性还是提高了，其实是降低了，所以异常处理能不用就不用，你不是写BUG的，只有在不可预测的逻辑性异常才要用
    '''

异常处理的实际使用

    try:
        print('----------------')
        l_list = [1, 2, 3]
        l_list[3]
        print('----------------')
        xxx
        print('----------------')
    except IndexError as l:
        print('异常以出现', l)
    '''
    ----------------
    异常以出现 list index out of range
    '''
# 从这里可以看出先报错l_list没有索引3的错误，但是报错后，后面的同级代码也自然不运行了
# 万一我们遇到不知道什么
    try:
        print('----------------')
        l_list = [1, 2, 3]
        # l_list[3]
        print('----------------')
        xxx
        print('----------------')
    except IndexError as l:
        print('异常以出现', l)
    except NameError as e:
        print('异常以出现', e)
    '''
    ----------------
    ----------------
    异常以出现 name 'xxx' is not defined
    '''
# 当不知道什么错的时候，后面可以加新的错误类型，但是这里也能看出来print是重复代码了，我们怎么解决呢
    except (IndexError, NameError) as l:
           print('异常以出现', l)
# 这样即可，那有没有万能异常解决的代码呢，什么类型都可以解决，有的
    except Exception as e:
        pass
# 然后我们加else finally 的用法
# else
    try:
        print('----------------')
        l_list = [1, 2, 3]
        # l_list[3]
        print('----------------')
        xxx
        print('----------------')
    # except (IndexError, NameError) as l:
    #     print('异常以出现', l)
    else:
        print('123')  # else与try无法直接连用，得通过except，else是这个是如果这个子代码块没有抛出异常，就会执行的代码
# finally
    try:
        print('----------------')
        l_list = [1, 2, 3]
        l_list[3]
        print('----------------')
        xxx
        print('----------------')
    # except (IndexError, NameError) as l:
    #     print('异常以出现', l)
    # else:
    #     print('123')
    finally:
        print('123')  # finally是可以能直接和try连用的，finally一般是会写直接调离系统的代码
